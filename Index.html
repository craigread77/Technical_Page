Introduction

HTML5 is the latest evolution of the standard that defines HTML. The term represents two different concepts. It is a new version of the language HTML, with new elements, attributes, and behaviors, and a larger set of technologies that allows the building of more diverse and powerful Web sites and applications. This set is sometimes called HTML5 & friends and often shortened to just HTML5.


<html>
    <head>
        <title>Hello World!</title>
    </head>
    <body>
        <h1>Hello World!</h1>
    </body>
</html>
Article

The article tag specified independant, self-contained content.

An article should make sense on its own and it should be possible to distribute independantly from the rest of the site.


<article>
    <h2>Florida Man wrestles aligator</h2>
    <p>Drunk florida man wrestles a full-grown ailgator.</p>
    <img src="florida-gator-man.jpg"> 
</article>
Details

The details tag specified additional details that the user can view or hide on demand. The content of a details element should not be visisble unless the open attribute is set.


<details>
    <summary>Copyright 1999-2018.</summary>
    <p> - by Refsnes Data. All Rights Reserved.</p>
    <p>All content and graphics on this web site are the property of the company Refsnes Data.</p>
</details>

As you can see from the example above, details can have pretty much anything nested inside of it.
Some good reasons to use the details tag are:


  • Increased accessibility without hindering the normal page experience.
  • Good habit to get used to new features.
  • Using new features is impressive.
  • Users can see more info without crowding the page.
  • Integrates well with javascript to show details in a new box upon click.
Figcaption

The figcaption tag defines a caption for a figure element.

The figcaption element can be placed as the first or last child of the figure element.


<figure>
    <img src="pic_trulli.jpg" alt="Trulli" style="width:100%">
    <figcaption>Fig.1 - Trulli, Puglia, Italy.</figcaption>
</figure>
Figure

The figure tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
While the content of the figure element is related to the main flow, its position is independent of the main flow, and if removed it should not affect the flow of the document.


<figure>
    <img src="img_pulpit.jpg" alt="The Pulpit Rock" width="304" height="228">
</figure>
Footer

A footer tag defines a footer for a document or section. A footer element should contain information about its containing element.


<footer>
    <p>Contact information: <a href="mailto:someone@example.com">someone@example.com</a></p>
</footer>
Header

A header element respresents a container for introductory content or a set of navigational links.
Typically contains one or more heading elements and a logo or icon.


<header>
    <img src="companylogo.png">
    <h1>MAXIMUS LLC</h1>
</header>
Main

The main tag specifies the main content of a document.

The content inside the main element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, nav links, copyright information, site logos, and search forms.


Note: There must not be more than one main element in a document. The main element must NOT be a descendant of an article, aside, footer, header, or nav element.


<main>
    <h1>Web Browsers</h1>
    <p>Google Chrome, Firefox, and Internet Explorer are the most used browsers today.</p>
    
    <article>
        <h1>Google Chrome</h1>
        <p>Google Chrome is a free, open-source web browser developed by Google,
        released in 2008.</p>
    </article>
    
    <article>
        <h1>Internet Explorer</h1>
        <p>Internet Explorer is a free web browser from Microsoft, released in 1995.</p>
    </article>
    
    <article>
        <h1>Mozilla Firefox</h1>
        <p>Firefox is a free, open-source web browser from Mozilla, released in 2004.</p>
    </article>
</main>
Mark

The mark tag defines highlighted or "marked" text. 


<p>Do not forget to buy <mark>milk</mark> today</p>
Nav

The nav tag defines a group of navigation links. This element is intended only for major blocks of navigation, not for any links on the site.


<nav>
    <a href="/html/">HTML</a>
    <a href="/css/">CSS</a>
    <a href="/js/">JavaScript</a>
    <a href="/jquery/">jQuery</a>
</nav>
Section

The section tag defines sections in a document, such as chapters, headers, footers, or any other document sections.


<section>
    <h1>WWF</h1>
    <p>The World Wide Fund for Nature (WWF) is....</p>
</section>
Summary

The summary tag defines a visible heading for the details element. The heading can be clicked to view/hide the details.


<details>
    <summary>Copyright 1999-2014.</summary>
    <p> - by Refsnes Data. All Rights Reserved.</p>
    <p>All content and graphics on this web site are the property of the company Refsnes Data.</p>
</details>
Time

The time tag defines a human-readable date/time.


This element can also be used to encode dates and times in a machine-readable way so that the user agents can offer to add birthday reminders or scheduled events to the user's calendar


<p>We open at <time>10:00</time> every morning.</p>
<p>I have a date on <time datetime="2008-02-14 20:00">Valentines day</time>.</p>
